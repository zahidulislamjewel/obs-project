## Conventions

| Convention | Value |
|---|---|
| Base path | `/api` |
| Content-Type | `application/json` |
| Date format | `YYYY-MM-DD` (ISO 8601) |
| Decimal format | JSON `number`, 2 decimal places for prices (e.g. `12.99`) |
| ID type | `long` (positive integer), auto-generated by backend |
| Pagination | 0-based page index (Spring default) |
| Error format | RFC 9457 `ProblemDetail` |
| Null optional fields | May be `null` or omitted in responses |

## CORS

The backend allows cross-origin requests from the frontend development origin.

**Allowed origin:** `http://localhost:4200`
**Allowed methods:** `GET`, `POST`, `PUT`, `DELETE`, `OPTIONS`
**Allowed headers:** `*`

> Implemented via `CorsConfig.java` (`@Configuration`).
> No CORS headers are needed in frontend code.

## Data Models

### BookResponse

Returned by all book read and write operations.

```json
{
  "id": 1,
  "title": "1984",
  "description": "A dystopian social science fiction novel.",
  "price": 12.99,
  "isbn": "978-0451524935",
  "publishedDate": "1949-06-08",
  "authorId": 1,
  "authorName": "George Orwell",
  "categoryId": 4,
  "categoryName": "Science Fiction",
  "stock": 150
}
```

| Field | Type | Nullable | Notes |
|---|---|---|---|
| `id` | `long` | no | Auto-generated primary key |
| `title` | `string` | no | |
| `description` | `string` | yes | |
| `price` | `number` | no | Decimal, always > 0 |
| `isbn` | `string` | no | Unique across all books |
| `publishedDate` | `string` | yes | Format: `YYYY-MM-DD` |
| `authorId` | `long` | no | FK to author |
| `authorName` | `string` | no | Denormalized from author entity |
| `categoryId` | `long` | no | FK to category |
| `categoryName` | `string` | no | Denormalized from category entity |
| `stock` | `int` | no | >= 0 |

### BookRequest

Request body for `POST /api/books` and `PUT /api/books/{id}`.

```json
{
  "title": "1984",
  "description": "A dystopian social science fiction novel.",
  "price": 12.99,
  "isbn": "978-0451524935",
  "publishedDate": "1949-06-08",
  "authorId": 1,
  "categoryId": 4,
  "stock": 150
}
```

| Field | Type | Required | Validation |
|---|---|---|---|
| `title` | `string` | yes | Must not be blank |
| `description` | `string` | no | |
| `price` | `number` | yes | Must be > 0 |
| `isbn` | `string` | yes | Must not be blank; must be unique |
| `publishedDate` | `string` | no | Format: `YYYY-MM-DD` |
| `authorId` | `long` | yes | Must reference an existing author |
| `categoryId` | `long` | yes | Must reference an existing category |
| `stock` | `int` | yes | Must be >= 0 |

> **IMPORTANT:** `authorId` and `categoryId` are **foreign keys**, not free-text strings.
> The frontend must load authors and categories first, then send their IDs.
> Sending a non-existent ID returns `404 Not Found`.

### AuthorResponse

Returned by `GET /api/authors`.

```json
{
  "id": 1,
  "name": "George Orwell",
  "bio": "English novelist and essayist."
}
```

| Field | Type | Nullable |
|---|---|---|
| `id` | `long` | no |
| `name` | `string` | no |
| `bio` | `string` | yes |

### CategoryResponse

Returned by `GET /api/categories`.

```json
{
  "id": 4,
  "name": "Science Fiction",
  "description": "Science fiction books."
}
```

| Field | Type | Nullable |
|---|---|---|
| `id` | `long` | no |
| `name` | `string` | no |
| `description` | `string` | yes |

### Page\<T\>

Returned by `GET /api/books`. Uses Spring Data's standard page wrapper.

```json
{
  "content": [ ],
  "totalPages": 2,
  "totalElements": 11,
  "size": 10,
  "number": 0,
  "first": true,
  "last": false,
  "numberOfElements": 10,
  "empty": false
}
```

> **Page index is 0-based.** Page 0 is the first page. `number` is the current page index.
> Frontend must use `response.content` to access the book array.

## Endpoints — Books

### GET /api/books

Returns a paginated list of all books.

**Query parameters**

| Parameter | Type | Default | Description |
|---|---|---|---|
| `page` | `int` | `0` | Zero-based page number |
| `size` | `int` | `10` | Items per page |
| `sort` | `string` | `id,asc` | Field and direction, e.g. `title,desc` |

**Request**

```http
GET /api/books?page=0&size=10&sort=title,asc
```

**Response — 200 OK**

```json
{
  "content": [
    {
      "id": 1,
      "title": "1984",
      "description": "A dystopian novel.",
      "price": 12.99,
      "isbn": "978-0451524935",
      "publishedDate": "1949-06-08",
      "authorId": 1,
      "authorName": "George Orwell",
      "categoryId": 4,
      "categoryName": "Science Fiction",
      "stock": 150
    }
  ],
  "totalPages": 2,
  "totalElements": 11,
  "size": 10,
  "number": 0,
  "first": true,
  "last": false
}
```

---

### GET /api/books/{id}

Returns a single book by ID.

**Path parameters**

| Parameter | Type | Description |
|---|---|---|
| `id` | `long` | Book ID |

**Response — 200 OK** — `BookResponse`

**Response — 404 Not Found** — see [Error Responses](#error-responses)

---

### POST /api/books

Creates a new book.

**Request body** — `BookRequest`

**Response — 201 Created** — `BookResponse`

**Response — 400 Bad Request** — validation failure (see [Error Responses](#error-responses))

**Response — 404 Not Found** — `authorId` or `categoryId` does not exist

---

### PUT /api/books/{id}

Fully replaces an existing book. All `BookRequest` fields apply.

**Path parameters**

| Parameter | Type | Description |
|---|---|---|
| `id` | `long` | ID of the book to update |

**Request body** — `BookRequest`

**Response — 200 OK** — `BookResponse`

**Response — 400 Bad Request** — validation failure

**Response — 404 Not Found** — book, author, or category ID does not exist

---

### DELETE /api/books/{id}

Deletes a book by ID.

**Path parameters**

| Parameter | Type | Description |
|---|---|---|
| `id` | `long` | ID of the book to delete |

**Response — 204 No Content** — empty body

**Response — 404 Not Found** — book ID does not exist

## Pagination

Spring Data's `Page<T>` is used for `GET /api/books`. Key facts:

| Detail | Value |
|---|---|
| Page index | **0-based** (first page = `?page=0`) |
| Default page size | `10` |
| Default sort | `id,asc` |
| Book array field | `content` (not `data`) |
| Total pages | `totalPages` |
| Total items | `totalElements` |
| Current page index | `number` |

**Frontend usage (Angular):**

```typescript
// Correct — use response.content to get books
this.http.get<Page<Book>>(`${base}/books`, { params })
  .pipe(map(res => res.content));
```

## Error Responses

All errors use [RFC 9457](https://www.rfc-editor.org/rfc/rfc9457) `ProblemDetail` format.

```json
{
  "type": "about:blank",
  "title": "Not Found",
  "status": 404,
  "detail": "Book not found with id: 99",
  "instance": "/api/books/99"
}
```

| Field | Type | Description |
|---|---|---|
| `type` | `string` | URI reference; `about:blank` if no dedicated type |
| `title` | `string` | Short human-readable summary |
| `status` | `int` | HTTP status code |
| `detail` | `string` | Human-readable explanation specific to this occurrence |
| `instance` | `string` | URI of the request that caused the error |

**Validation error (400)** — `detail` contains all violated field constraints, comma-separated:

```json
{
  "type": "about:blank",
  "title": "Bad Request",
  "status": 400,
  "detail": "title: Title is required, price: Price must be greater than 0",
  "instance": "/api/books"
}
```

## HTTP Status Code Summary

| Code | When |
|---|---|
| 200 | Successful GET or PUT |
| 201 | Successful POST (book created) |
| 204 | Successful DELETE (no body) |
| 400 | Validation errors in request body |
| 404 | Book, author, or category ID not found |
| 500 | Unhandled exception (e.g. duplicate ISBN violates DB constraint) |

## Known Integration Pitfalls

These are the most common causes of integration bugs in this project.

### 1. Author and Category are foreign keys, not strings

The frontend must send `authorId: number` and `categoryId: number` in `BookRequest`.
Sending plain strings (`"author": "George Orwell"`) will cause a `400 Bad Request`.

The correct flow for the book form:
1. Load authors via `GET /api/authors`
2. Load categories via `GET /api/categories`
3. Populate dropdowns; bind selected item's `id`
4. Submit `BookRequest` with `authorId` and `categoryId`

### 2. Pagination is 0-based

The backend uses Spring's default 0-based page index.
- First page: `?page=0` (not `?page=1`)
- The book array is at `response.content` (not `response.data`)

### 3. ISBN must be unique

Submitting a duplicate ISBN does not return `400` — it throws a DB constraint violation resulting in `500`.
Frontend should surface this as a generic server error and advise the user to check the ISBN.

### 4. Error format is ProblemDetail, not a custom envelope

The backend returns `{ type, title, status, detail, instance }`.
It does **not** return `{ status, error, message, timestamp, path }`.
Angular HTTP interceptors or error handlers must read from `error.detail`, not `error.message`.

### 5. Base path has no version prefix

All endpoints are under `/api/*`, not `/api/v1/*`.
Correct: `http://localhost:8080/api/books`
Wrong: `http://localhost:8080/api/v1/books`

### 6. CORS is enabled for localhost:4200 only

In production or Docker environments, the allowed origin must be updated to match the actual frontend URL.
The current `CorsConfig.java` hardcodes `http://localhost:4200`.
