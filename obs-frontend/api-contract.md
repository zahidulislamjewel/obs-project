## Overview

This document defines the REST API contract between the OBS frontend and backend.
The frontend currently uses a mock service (`book.service.ts`) that mirrors these
contracts exactly — replacing `of(...)` with real HTTP calls is all that's needed
during backend integration.

All requests and responses use **JSON** (`Content-Type: application/json`).

---

## Conventions

| Convention | Value |
|---|---|
| Base path | `/api/v1` |
| Date format | `YYYY-MM-DD` (ISO 8601) |
| Decimal format | 2 decimal places for prices (e.g. `35.99`) |
| ID type | Positive integer, auto-generated by backend |
| Null fields | Omit optional fields rather than sending `null` |
| Encoding | UTF-8 |

## Data Models

### Book

The core resource of the API.

```json
{
  "id": 1,
  "title": "Clean Code",
  "description": "A handbook of agile software craftsmanship.",
  "price": 35.99,
  "originalPrice": 49.99,
  "author": "Robert C. Martin",
  "category": "Programming",
  "publishedDate": "2008-08-01",
  "stock": 20,
  "coverUrl": "https://covers.openlibrary.org/b/isbn/9780132350884-L.jpg",
  "rating": 4.8,
  "reviewCount": 2341,
  "badge": "bestseller"
}
```

#### Field Reference

| Field | Type | Required | Description |
|---|---|---|---|
| `id` | `integer` | Read-only | Auto-generated unique identifier |
| `title` | `string` | Yes | Book title. Max 255 chars |
| `description` | `string` | No | Full book description. Max 2000 chars |
| `price` | `number` | Yes | Current sale price. Must be ≥ 0 |
| `originalPrice` | `number` | No | Pre-discount price. Must be > `price` if present |
| `author` | `string` | Yes | Author name(s). Max 255 chars |
| `category` | `string` | Yes | One of the allowed categories (see [Categories](#categories)) |
| `publishedDate` | `string` | Yes | Publication date in `YYYY-MM-DD` format |
| `stock` | `integer` | Yes | Available inventory count. Must be ≥ 0 |
| `coverUrl` | `string` | No | Absolute URL to cover image |
| `rating` | `number` | No | Average rating 0.0–5.0 (1 decimal). Computed by backend |
| `reviewCount` | `integer` | No | Total number of reviews. Computed by backend |
| `badge` | `string` | No | Promotional badge. Enum: `bestseller` \| `new` \| `sale` |

### BookCreateRequest

Request body for creating a book. Excludes read-only/computed fields.

```json
{
  "title": "Clean Code",
  "description": "A handbook of agile software craftsmanship.",
  "price": 35.99,
  "originalPrice": 49.99,
  "author": "Robert C. Martin",
  "category": "Programming",
  "publishedDate": "2008-08-01",
  "stock": 20,
  "coverUrl": "https://covers.openlibrary.org/b/isbn/9780132350884-L.jpg",
  "badge": "bestseller"
}
```

### BookUpdateRequest

Same shape as `BookCreateRequest`. All fields are optional — only provided
fields are updated (PATCH semantics on PUT).

### PaginatedResponse\<T\>

```json
{
  "data": [...],
  "page": 1,
  "pageSize": 20,
  "totalItems": 12,
  "totalPages": 1
}
```

### ErrorResponse

```json
{
  "status": 404,
  "error": "Not Found",
  "message": "Book with id 99 not found",
  "timestamp": "2026-03-07T14:32:00Z",
  "path": "/api/v1/books/99"
}
```

## Endpoints

### Books

#### `GET /books` — List all books

Returns a paginated list of books. Supports optional filtering and sorting.

**Query Parameters**

| Parameter | Type | Default | Description |
|---|---|---|---|
| `page` | integer | `1` | Page number (1-based) |
| `pageSize` | integer | `20` | Results per page. Max `100` |
| `category` | string | — | Filter by category (exact match) |
| `badge` | string | — | Filter by badge: `bestseller`, `new`, `sale` |
| `search` | string | — | Full-text search on title and author |
| `sort` | string | `id` | Sort field: `id`, `title`, `price`, `rating`, `publishedDate` |
| `order` | string | `asc` | Sort direction: `asc`, `desc` |
| `minPrice` | number | — | Minimum price filter |
| `maxPrice` | number | — | Maximum price filter |

**Request**
```
GET /api/v1/books?category=Programming&sort=rating&order=desc&page=1&pageSize=20
```

**Response `200 OK`**
```json
{
  "data": [
    {
      "id": 1,
      "title": "Clean Code",
      "description": "A handbook of agile software craftsmanship.",
      "price": 35.99,
      "author": "Robert C. Martin",
      "category": "Programming",
      "publishedDate": "2008-08-01",
      "stock": 20,
      "coverUrl": "https://covers.openlibrary.org/b/isbn/9780132350884-L.jpg",
      "rating": 4.8,
      "reviewCount": 2341,
      "badge": "bestseller"
    }
  ],
  "page": 1,
  "pageSize": 20,
  "totalItems": 12,
  "totalPages": 1
}
```

## Error Responses

All errors follow a consistent `ErrorResponse` envelope:

```json
{
  "status": 422,
  "error": "Unprocessable Entity",
  "message": "originalPrice must be greater than price",
  "timestamp": "2026-03-07T14:32:00Z",
  "path": "/api/v1/books"
}
```

For validation errors, an additional `violations` array is included (see `POST /books` example above).

---

## HTTP Status Codes

| Code | Meaning | When Used |
|---|---|---|
| `200 OK` | Success | GET, PUT success |
| `201 Created` | Resource created | POST success |
| `204 No Content` | Success, no body | DELETE success |
| `400 Bad Request` | Validation error | Invalid request body fields |
| `404 Not Found` | Resource missing | Book ID does not exist |
| `409 Conflict` | Duplicate resource | Creating a book that already exists (by title + author) |
| `422 Unprocessable Entity` | Business rule violation | e.g. `originalPrice` ≤ `price` |
| `500 Internal Server Error` | Server failure | Unexpected backend error |

## Frontend Integration Notes

When the backend is ready, update `book.service.ts` to use `HttpClient` instead of `of()`:

```typescript
// Before (mock)
getBooks(): Observable<Book[]> {
  return of(this.books);
}

// After (real API)
private readonly API = 'http://localhost:8080/api/v1';

getBooks(): Observable<Book[]> {
  return this.http.get<{ data: Book[] }>(`${this.API}/books`)
    .pipe(map(res => res.data));
}

getBook(id: number): Observable<Book> {
  return this.http.get<Book>(`${this.API}/books/${id}`);
}

addBook(book: Omit<Book, 'id'>): Observable<Book> {
  return this.http.post<Book>(`${this.API}/books`, book);
}

updateBook(book: Book): Observable<Book> {
  return this.http.put<Book>(`${this.API}/books/${book.id}`, book);
}

deleteBook(id: number): Observable<void> {
  return this.http.delete<void>(`${this.API}/books/${id}`);
}

getCategories(): Observable<string[]> {
  return this.http.get<string[]>(`${this.API}/books/categories`);
}
```

**CORS:** The backend must allow `http://localhost:4200` during development.

**Environment:** Use Angular environment files (`environment.ts` / `environment.prod.ts`) for the base URL:
```typescript
export const environment = {
  production: false,
  apiUrl: 'http://localhost:8080/api/v1'
};
```
